  1. Other readings:
    • Object Methods: this.
      • Object represent entities of the real world.
      • Here's an object literal: user = { sayHi() { alert( this.userName ); } };
      • Object's actions are represented in JS by functions stored as object properties.
        • For example, we can use a Function Expression to create a function and assign it to the property user instance as user.sayHi = function() { ... };, and then make a call to it as user.sayHi();.
      • A function that is a property of an object is called its method.
      • The value of this is the object instance “before the dot”, the one used to call the method, or the one that called the method. user.sayHi(); prints out the user name.
        • Methods allow objects to “act” like object.doSomething()
      • In JavaScript, keyword this behaves unlike most other programming languages. It can be used in any function, even if it’s not a method of an object. The value of this is evaluated during the run-time, depending on the context.
      • The concept of run-time evaluated this has both pluses and minuses. On the one hand, a function can be reused for different objects. On the other hand, the greater flexibility creates more possibilities for mistakes.
        • When a function is declared, it may use this, but that this has no value until the function is called.
        • A function can be copied between objects.
        • When a function is called in the “method” syntax: object.method(), the value of this during the call is object.
        • Note that arrow functions are special: they have no this. When this is accessed inside an arrow function, it is taken from outside.
      • Example on chaining: 
        
let ladder = { step: 0,
	up() {
		this.step++;
		return this;
	},
	down() {
		this.step--;
		return this;
	},
	showStep() {
		alert( this.step );
		return this;
	}
};

ladder.up().up().down().up().down().showStep();
